## Verdict

- Per-regime scale improves held-out log loss over base_rate/tau alone: **False** (0.5449 -> 0.5498).

> Adding a per-regime lambda scale does **not** help — held-out log loss rises. On the full data every regime's best scale is ~1.0, meaning the base multipliers (pressure, score, time, cards) already encode what the regime label would add; the extra per-regime freedom just overfits the training folds. This is exactly Section 24's warning that thinly-populated regimes refit noise, and it confirms the regime detector's value is in *interpretation* (panel, confidence), not as an extra goal-rate multiplier for this target.
